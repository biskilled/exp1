_Last updated: 2026-04-27_

## Commands

| Command | Trigger | LLM | Output |
|---------|---------|-----|--------|
| `/memory` | Manual or Stop hook | Haiku (project_synthesis) | Writes all context files from project_state.json + DB |
| `/push` | Manual or Stop hook | Sonnet (commit_analysis) + Haiku (commit_message, commit_symbol×N) | Stores commit in DB; updates code.md in background |
| `/wi classify` | Manual (or threshold, opt-in) | Haiku (classification) | Creates draft work items in mem_work_items |
| `/wi approve` | Manual (UI click) | OpenAI text-embedding-3-small | Assigns real ID, computes embedding |
| `/wi pipeline` | Manual (MCP or API) | Sonnet ×4 (PM→Arch→Dev→Review) | Writes acceptance_criteria + implementation_plan to item |
| `/search` | Manual | — (pgvector cosine) | Returns similar work items |
| `/embed` | Manual | OpenAI text-embedding-3-small | Embeds unapproved items for search |
| Session start hook | Auto (Claude Code Start) | — | Refreshes CLAUDE.md from cached project_state.json |
| Session stop hook | Auto (Claude Code Stop) | Same as /push + /memory | commit-push → extract symbols → /memory |

## Memory Files (written by `/memory`)

All files live in `workspace/{project}/memory/`. Source = `project_state.json` + DB.

| File | LLM to generate | Source data |
|------|----------------|-------------|
| `claude/CLAUDE.md` | None (render) | project_state.json (tech_stack, key_decisions, in_progress) + mem_work_items (active) + mem_mrr_commits_code (recently changed) + mem_mrr_commits_file_stats (hotspots) |
| `cursor/rules.md` | None (render) | project_state.json subset (stack + decisions) + active work items |
| `openai/compact.md` | None (render) | project_state.json + work items (condensed) |
| `openai/full.md` | None (render) | project_state.json + work items (full) |
| `context.md` | None (render) | project_state.json + tech stack (injected by CLI) |
| `code.md` | None (render) | mem_mrr_commits_code (recently changed symbols) + mem_mrr_commits_file_stats (hotspots) + mem_mrr_commits_file_coupling (coupling) |
| `project_state.json` | **Haiku** (project_synthesis) | Last 50 prompts from DB + PROJECT.md sections + prior state |

## project_state.json (the hub)

Generated by `/memory` → Haiku reads recent prompts + PROJECT.md → writes JSON.
All context files are rendered FROM this file (no direct LLM during render).

```
Fields used by renderer:
  tech_stack       → CLAUDE.md "Stack & Architecture"
  key_decisions    → CLAUDE.md "Key Architectural Decisions"
  in_progress      → CLAUDE.md "In Progress"
  project_summary  → CLAUDE.md header (short description)
```

PROJECT.md is **authoritative**: sections Vision / Core Goals / Memory Architecture
feed the Haiku synthesis prompt. If PROJECT.md contradicts project_state.json,
PROJECT.md wins on next `/memory` run.

## Work Items Flow

```
mem_mrr_* (prompts/commits) → /wi classify [Haiku] → draft items (wi_id=AI0001)
       ↓ user approves
approved item (wi_id=US/FE/BU/TA) → embedding computed [OpenAI]
       ↓ manual trigger
/wi pipeline [Sonnet×4] → acceptance_criteria + implementation_plan saved to item
```

**IDs**: `AI*` = draft (unreviewed) | `US/FE/BU/TA/PO/RE` = approved | `REJ*` = rejected

## Embedding Strategy

| What | Model | When | Table |
|------|-------|------|-------|
| Approved work items | OpenAI text-embedding-3-small | On approve | `mem_work_items.embedding VECTOR(1536)` |
| Unapproved items | OpenAI text-embedding-3-small | Manual `/embed` | `mem_work_items.embedding` |
| Prompts/commits | Not embedded by default | Never (manual only) | — |

Search = `POST /search/semantic` → pgvector cosine similarity over `mem_work_items.embedding`.

## How New Requirements Override Old Ones

1. Edit `PROJECT.md` — update Memory Architecture / Vision / Core Goals
2. Run `/memory` — Haiku reads PROJECT.md as authoritative, replaces stale `key_decisions`
3. All context files (CLAUDE.md, cursorrules, etc.) re-rendered from updated `project_state.json`

Between `/memory` runs, CLAUDE.md reflects the last cached `project_state.json`.
