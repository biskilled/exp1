# FastAPI Feature Design

Design the requested FastAPI feature. Output a structured document:

1. **Endpoint(s)**: method, path, request body, response model
2. **Pydantic models**: new models needed (fields + types)
3. **Business logic**: what goes in core/, what goes in the router
4. **Database changes**: new tables/columns if needed
5. **Tests**: what to test, happy path + error cases
6. **Implementation order**: numbered steps
