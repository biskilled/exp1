# Memory Output Templates

These files define the format and sections for each LLM-specific context file
generated by `POST /projects/{project}/memory` and `memory_files.py`.

## Files generated per project

| File | Location | Consumer |
|------|----------|---------|
| `CLAUDE.md` | `_system/claude/CLAUDE.md` + `{code_dir}/CLAUDE.md` | Claude Code / Claude CLI (auto-loaded) |
| `MEMORY.md` | `_system/claude/MEMORY.md` + `{code_dir}/MEMORY.md` | Claude Code reads via CLAUDE.md reference |
| `cursor/rules.md` | `_system/cursor/rules.md` + `.cursor/rules/aicli.mdrules` | Cursor AI |
| `aicli/context.md` | `_system/aicli/context.md` | aicli CLI — injected as system block for all providers |
| `aicli/copilot.md` | `_system/aicli/copilot.md` + `.github/copilot-instructions.md` | GitHub Copilot |
| `llm_prompts/compact.md` | `_system/llm_prompts/compact.md` | GPT-4 / small-window direct API (<2000 tokens) |
| `llm_prompts/full.md` | `_system/llm_prompts/full.md` | Claude / DeepSeek / Gemini direct API (full context) |
| `llm_prompts/gemini_context.md` | `_system/llm_prompts/gemini_context.md` | Gemini Files API upload |
| `llm_prompts/openai.md` | `_system/llm_prompts/openai.md` | OpenAI API / Codex (system message) |

## Template variables (available in all templates)

```
{project}         — project slug
{ts}              — generation timestamp "YYYY-MM-DD HH:MM UTC"
{facts.*}         — project facts by category (stack, pattern, convention, constraint, client)
{active_tags}     — list of open/active planner tags
{key_decisions}   — LLM-extracted stable decisions (from project_state.json)
{tech_stack}      — LLM-extracted tech stack dict (from project_state.json)
{in_progress}     — LLM-extracted in-progress items
{project_summary} — LLM-extracted 2-3 sentence summary
{project_md_60}   — first 60 lines of PROJECT.md
```

## Refresh triggers

- **Automatic**: tag or fact changed → `write_root_files()` runs (CLAUDE.md + llm_prompts only)
- **Manual**: `POST /projects/{project}/memory` or `/memory` command in aicli CLI
  → `generate_memory()` runs (all files including MEMORY.md)

## Multi-LLM support

For each LLM tool:

| Tool | File to use | How to load |
|------|------------|-------------|
| Claude Code | `CLAUDE.md` | Auto-loaded from code root |
| Claude CLI | `CLAUDE.md` + `MEMORY.md` | Auto-loaded |
| Cursor | `.cursor/rules/aicli.mdrules` | Cursor reads this automatically |
| GitHub Copilot | `.github/copilot-instructions.md` | Copilot reads this automatically |
| OpenAI / GPT-4 | `llm_prompts/compact.md` | Pass as system message |
| Claude API | `llm_prompts/full.md` | Pass as system message |
| DeepSeek | `llm_prompts/full.md` | Pass as system message |
| Gemini | `llm_prompts/gemini_context.md` | Upload via Files API |
| Codex CLI | `llm_prompts/openai.md` | Pass as `--system-prompt` |
| MCP tool | `GET /memory/{project}/llm-prompt?format=full` | Backend serves on demand |
