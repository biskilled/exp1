[PROJECT CONTEXT: aicli]
aicli provides unified LLM memory across Claude Code, CLI, Cursor, and web UI—shared project context, decisions, and feature history. The platform combines a 3-layer memory architecture (raw captures → structured artifacts → approved work items), role-based agents with parameter overrides, async multi-LLM DAG pipelines, and semantic search via pgvector. Current focus is UI execution flows (pipeline & role direct execution with unified input bar) and dashboard restoration with approved vs pending work items tracking.
Stack: language_cli=Python 3.12, cli_framework=prompt_toolkit + rich, backend_framework=FastAPI + uvicorn, backend_auth=JWT (python-jose + bcrypt) + DEV_MODE toggle, database=PostgreSQL 15+ with pgvector (1536-dim, text-embedding-3-small)
