[PROJECT CONTEXT: aicli]
aicli is a unified LLM memory system that keeps Claude Code, Cursor, aicli CLI, and web UI synchronized across a shared PostgreSQL database. It auto-captures commits, prompts, and session context; synthesizes project state via Claude Haiku batched every N prompts; manages work items (use cases/features/bugs/tasks) with AI classification, user approval, pgvector embedding, and a 4-agent DAG pipeline; and generates five managed output files (PROJECT.md, CODE.md, CLAUDE.md, cursor/rules, api/) from a single project_state.json source. Latest focus: code optimization (CTEs bounded, N+1 eliminated, file sizes normalized) and embedding strategy clarity (approved items only).
Stack: cli=Python 3.12 + prompt_toolkit + rich, backend=FastAPI + uvicorn + python-jose + bcrypt + psycopg2, frontend=Vanilla JS + Electron + Vite, ui_components=xterm.js + Monaco editor + Cytoscape.js, storage=PostgreSQL 15+ with pgvector (1536-dim, text-embedding-3-small)
In progress: Code quality audit (sessions 40+): verified all modules <1500 lines, SQL queries optimized (no N+1 or unbounded CTEs), token counting uses len(text)//4, _load_context() split into _query_db_into_ctx() and _parse_project_md(), date cascade validation enforced., Memory file convergence: unified get_project_context() path for both /memory POST and commit/work-item triggers; all 5 output files use same DB source and Haiku synthesis; removed duplicate render paths., Work item approval flow: unapproved drafts (AI prefix) never embed or run pipeline; once user approves, wi_id changes to UC/FE/BU/TA, embedding is computed, and 4-agent pipeline triggers automatically.
Decisions: Memory architecture: 3 layers — raw captures (mem_mrr_*), structured artifacts (mem_ai_project_facts), and work items (mem_work_items) with pgvector embeddings for approved items only (1536-dim, text-embedding-3-small).; Work items: unified hierarchy with wi_parent_id linking features/bugs/tasks to use_case parents; approved items (wi_id: UC/FE/BU/TA) embed and trigger 4-agent pipeline; unapproved drafts (AI prefix) never embed or run pipeline.; File generation: /memory POST endpoint is the ONLY writer to project_state.json via get_project_context() + Haiku synthesis; this single JSON drives all 5 output files (CLAUDE.md, CODE.md, PROJECT.md, cursor/rules, api/).
