## Key Decisions

- Workspace structure simplified: aicli/cli/{claude,mcp}/ for hooks/configs; pipelines/{prompts,samples}/ for workflows; documents/ for project files; state/ for runtime state
- .ai/ and _system/ folders removed; all generated files now under aicli/cli/ with templates in backend/memory/templates/
- Dual storage: PostgreSQL 15+ with pgvector (1536-dim) for semantic search; unified mem_mrr_* and mem_work_items tables
- Memory files (PROJECT.md, CODE.md, CLAUDE.md, cursorrules.md, recent_work.md) generated by /memory endpoint from project_state.json + database; token-limited by project.yaml config
- backend/memory/memory.yaml is canonical single source for file mapping (not duplicated); templates in backend/memory/templates/
- Code.md structure: public symbols (classes/methods/functions) + file coupling/hotspot tables with is_latest BOOLEAN pattern; generated from mem_mrr_commits_code on each /memory run
- mem_mrr_commits_code (19 columns) with is_latest BOOLEAN: replaces separate code_symbols table; updated per commit; partial index optimization
- Project facts deprecated from automatic /memory generation; facts extracted inline from memory synthesis; conflict_detection merged into mem_project_state.yaml
- Work Items + Use Cases: items tab shows pending AI-classified items; use cases tab displays hierarchy with due dates; drag-and-drop unified for both with document.elementFromPoint() target detection
- Undo as persistent toolbar button: stores reverse API call closure capturing original parent_id before link modification
- JWT authentication: python-jose + bcrypt with DEV_MODE toggle; hierarchical Clients → Users → Projects
- LLM provider adapters: Claude/OpenAI/DeepSeek/Gemini/Grok as independent modules in agents/providers/ with send(prompt, system) → str contract
- All backend LLM prompts migrated to YAML files under backend/memory/mem_*.yaml (named by table domain); templates in backend/memory/templates/
- Embedding strategy: mem_work_items VECTOR(1536) used for /search/semantic; mem_mrr_commits and mem_mrr_prompts embeddings optional for MCP semantic history search
- MCP tools rewired: create_entity→POST /wi/{project}, list_work_items→GET /wi/{project}, run_work_item_pipeline uses mem_work_items not entities
