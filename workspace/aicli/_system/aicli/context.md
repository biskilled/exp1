[PROJECT CONTEXT: aicli]
**aicli** is a shared AI memory platform that unifies conversation history, commits, and structured metadata across multiple AI interfaces (Claude CLI, Cursor, browser, etc.). It uses flat-file JSONL storage as primary (with PostgreSQL+pgvector for semantic search), provides a desktop Electron UI with xterm.js terminal + Monaco editor, and enables developers to tag sessions with nested categories, link commits to prompts, and synthesize memory via Claude Haiku. Currently at v2.2.0, fully supporting history rotation, bidirectional commit-prompt linking, zero-DB-call tag caching, and always-on AI suggestions.
Stack: cli=Python 3.12 + prompt_toolkit + rich, backend=FastAPI + uvicorn + python-jose + bcrypt, frontend=Vanilla JS (no framework, no bundler) + Electron shell + Vite dev server, ui_components=xterm.js (embedded terminal) + Monaco editor + Cytoscape.js (graph flows), storage_primary=JSONL (history.jsonl with rotation support, commit_log.jsonl), JSON, CSV — flat file first
In progress: History rotation fully operational — /memory triggers rotation at configurable row threshold (default 500); creates timestamped archive (history_YYMMDDHHSS), original becomes history.jsonl (2026-03-14), Commit-to-prompt bidirectional linking matured — source_id timestamp from history.jsonl maps to commits via POST /entities/events/tag-by-source-id; multiple commits per session each tagged to originating prompt (2026-03-14), Tag cache persistence in history view — all categories/values loaded once on history tab open via Promise.all; color preservation on save prevents DB thrashing; zero DB calls during tag picker (2026-03-14)
