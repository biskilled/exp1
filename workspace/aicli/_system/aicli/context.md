[PROJECT CONTEXT: aicli]
aicli is a shared AI memory platform for software development that maintains persistent context across sessions by storing project history in PostgreSQL with semantic embeddings. It separates development (backend FastAPI + Python CLI, frontend Electron + Vanilla JS) and enables work item lifecycle management with AI-generated metadata, use case synthesis, and workflow automation via async DAG execution. Current focus is stabilizing Work Items/Use Cases UI separation, MD file generation accuracy with due date enforcement, and hook-log reliability for prompt persistence.
Stack: cli=Python 3.12 + prompt_toolkit + rich, backend=FastAPI + uvicorn + python-jose + bcrypt + psycopg2, frontend=Vanilla JS + Electron shell + Vite dev server, ui_components=xterm.js + Monaco editor + Cytoscape.js + cytoscape-dagre, storage_primary=PostgreSQL 15+ with pgvector (1536-dim, text-embedding-3-small)
In progress: MD file generation alignment: Use Cases now generate MD files with correct item categorization by type (bug/feature/task), proper status counts using recursive CTEs to capture all descendants, and completed/in-progress/open item sections, Use Case due date management: Due dates can be set as calendar dates or day offsets (e.g., 8 days or 05/05/26); connected items must finish by parent due date with auto-resolution of re-parenting conflicts, Session ID and timestamp visibility: Chat and History tabs now display last 5 chars of session ID in header with full UUID available on click; timestamps formatted as YY/MM/DD-HH:MM next to user prompts
