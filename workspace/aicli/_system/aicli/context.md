[PROJECT CONTEXT: aicli]
aicli is a shared AI memory platform that integrates development workflows with multi-layer semantic memory (ephemeral → raw capture → LLM digests → work items/features). It combines a Python 3.12 CLI, FastAPI backend with PostgreSQL/pgvector, and an Electron desktop UI featuring async DAG workflow execution, multi-provider LLM support (Claude/OpenAI/DeepSeek/Gemini/Grok), and intelligent code chunking. Currently in active refinement with database schema standardization complete, session history UI fully functional, and dashboard/pipeline visualization operational.
Stack: cli=Python 3.12 + prompt_toolkit + rich, backend=FastAPI + uvicorn + python-jose + bcrypt + psycopg2, frontend=Vanilla JS + Electron shell + Vite dev server, ui_components=xterm.js + Monaco editor + Cytoscape.js + cytoscape-dagre, storage_primary=PostgreSQL 15+ with pgvector (1536-dim, text-embedding-3-small)
In progress: Database schema refactor (m051-m052): user_id migrated from UUID to SERIAL INT; all 18 tables reordered to canonical form; updated_at added to mirror tables, Session history UI improvements: Chat and History tabs display sessions with source badge, phase chip, session ID, and timestamp; stale session loading fixed, Hook-log endpoint verification: all 531 prompts (389 DB + ~142 JSONL merged) loading correctly with proper sort order and session attribution
