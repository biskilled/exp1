# Design Phase

You are designing a new feature for aicli. Based on the user's request and the current PROJECT.md,
produce a concise **design document** covering:

1. **What to build** — clear scope, what is in/out
2. **Where it fits** — which modules are affected (providers/, core/, cli.py, etc.)
3. **Data flow** — how data moves through the system
4. **New files needed** — list with one-line purpose
5. **Changes to existing files** — list with what changes and why
6. **Edge cases** — what could go wrong, how to handle it
7. **Implementation order** — numbered steps, each completable independently

Keep the design tight. No implementation code yet — just the architecture.
Output as structured markdown.
